## 1. 产品简介

### 1.1 产品定位

**HHS（Health Hack System）** 是一款面向个人用户的智能健康管理平台，集健康数据追踪、AI 智能分析、实时监控、预警通知和可穿戴设备联动于一体，帮助用户全面掌握自身健康状况。

### 1.2 核心功能一览

| 功能模块 | 说明 | 入口路径 |
|---------|------|---------|
| 仪表盘 | 健康数据综合概览与快捷操作 | 首页 `/dashboard` |
| 智能录入 | 快捷/AI/OCR/设备同步四种录入方式 | `/data-input` |
| 健康指标 | 历史指标查询、编辑与趋势分析 | `/health/metrics` |
| 实时监控 | 实时指标录入与动态趋势图 | `/realtime/monitor` |
| 健康评分 | 多维度综合健康评估与 AI 健康报告 | `/health/score` |
| 智能预警 | 多级预警通知、统计分析与个性化阈值 | `/health/alerts` |
| AI 顾问 | 智能健康咨询对话（页面+浮动助手） | `/ai/chat` |
| 预防保健 | 健康档案、保健指标与风险评估 | `/prevention/profile` |
| 保健指标 | 睡眠、运动、心情等日常保健追踪 | `/wellness/dashboard` |
| 体检报告 | 报告管理与异常追踪 | `/screening/list` |
| 个人中心 | 个人资料、头像上传与推送设置 | `/user/profile` |

### 1.3 系统架构概览

```
┌───────────────────────────────────────────────────────┐
│                    用户（浏览器/移动端）                │
└─────────────────────┬─────────────────────────────────┘
                      │ HTTPS / WebSocket
┌─────────────────────▼─────────────────────────────────┐
│              Vue 3 前端 (Element Plus UI)              │
│         ┌── 浮动 AI 助手（全局可用）──┐               │
│         └── 暗/亮主题自适应 ──────────┘               │
├───────────────────────────────────────────────────────┤
│              Spring Boot 后端 (REST API)               │
├────────┬────────┬────────┬────────┬──────────────────┤
│ MySQL  │ Redis  │ 通义千问 │ 百度OCR │ 华为/小米 OAuth │
└────────┴────────┴────────┴────────┴──────────────────┘
```

## 2. 系统要求与准备

### 2.1 浏览器要求

| 浏览器 | 最低版本 | 推荐版本 |
|--------|---------|---------|

- 稳定的互联网连接

- **穿戴设备**: 华为手环/手表、小米手环/手表（可选）

## 3. 账户管理

### 3.1 注册新账户

**操作路径**: 系统首页 → 点击「立即注册」

**步骤**:

1. 填写注册信息：

   | 字段 | 要求 | 说明 |
   |------|------|------|
   | 用户名 | 4-32个字符 | 用于登录，注册后不可修改 |
   | 密码 | 6-64个字符 | 建议包含大小写字母、数字和特殊字符 |
   | 确认密码 | 与密码一致 | 再次输入确认 |
   | 邮箱 | 可选 | 格式如 `user@example.com`，用于通知 |

2. 点击「立即注册」完成注册
3. 注册成功后自动跳转至登录页

> **注意**: 用户名一旦注册不可修改。请妥善保管密码，建议使用强密码。

### 3.2 登录系统

**操作路径**: 系统首页

1. 输入用户名和密码
2. 点击「登录」按钮
3. 登录成功后进入仪表盘

**会话说明**:
- 登录凭证有效期为 **7天**，期间无需重复登录
- Token 过期后需重新登录
- 同一账户不支持多端同时在线

### 3.3 退出登录

**操作路径**: 右上角用户头像 → 退出登录

退出后系统清除本地登录凭证，需重新登录。

**操作路径**: 登录后默认进入 `/dashboard`

仪表盘是系统的信息中枢，提供健康状况一览与快捷操作入口。

### 4.1 页面布局

```
┌─────────────────────────────────────────────────────┐
│  顶部区域                                            │
│  [欢迎回来，用户昵称]     [添加指标] [AI咨询]         │
├─────────────────────────────────────────────────────┤

│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │

```

### 4.2 统计卡片说明

| 卡片 | 说明 | 点击行为 |
|------|------|---------|
| 健康评分 | 综合健康评估分数（0-100），含评分等级和变化趋势标识 | 跳转至健康评分页 |
| 未读预警 | 待处理的健康预警数量，有未读预警时显示红色圆点动画 | 跳转至预警列表页 |
| 健康指标 | 已记录的健康数据总条数 | 跳转至健康指标页 |
| 体检报告 | 已上传的体检报告数量 | 跳转至筛查报告页 |

### 4.3 健康评分卡片

左侧展示环形评分图（使用 `HealthScoreCircle` 组件），包含：
- 综合评分数值（0-100）
- 评分等级标签（优秀/良好/一般/需改善）
- 评分提示文字
- 评分因素标签（如 BMI、血压、睡眠等影响因素）

无评分数据时显示引导信息，引导用户添加健康数据。

### 4.4 最近预警卡片

右侧展示最近 5 条预警记录，每条包含：
- 左侧色条：按预警级别着色（红色-严重、黄色-警告、绿色-恢复等）
- 预警标题和时间
- 预警描述（最多显示两行）
- 无预警时显示绿色勾选图标

### 4.5 快速入口

水平滚动的一组快捷入口按钮，包含：
- **添加指标**: 跳转至健康指标页
- **AI 顾问**: 跳转至 AI 健康咨询对话
- **上传报告**: 跳转至筛查报告页
- **实时监控**: 跳转至实时健康监控
- **保健中心**: 跳转至保健指标仪表盘

## 5. 健康数据录入

**操作路径**: 左侧导航栏 → 智能录入 (`/data-input`)

HHS 提供 **四种** 数据录入方式，覆盖不同场景需求：

| 录入方式 | 适用场景 | 便捷度 | 准确度 |
|---------|---------|--------|--------|
| 快捷录入 | 手动输入单个指标 | ★★★ | ★★★★★ |
| AI 智能录入 | 自然语言批量描述 | ★★★★★ | ★★★★ |
| OCR 识别 | 纸质报告/化验单拍照 | ★★★★ | ★★★ |
| 设备同步 | 可穿戴设备自动导入 | ★★★★★ | ★★★★★ |

### 5.1 快捷录入

#### 5.1.1 操作步骤

1. 进入「智能录入」页面 → 点击「快捷录入」卡片
2. 点击需要录入的指标按钮（如：血糖、心率、血压等）
3. 在弹出的对话框中：
   - 输入数值（系统实时验证范围）
   - 选择记录日期（默认当天，支持补录历史数据）
4. 点击「保存」完成录入

#### 5.1.2 支持的指标

**健康指标**:

| 指标 | 单位 | 有效范围 | 精度 |
|------|------|---------|------|
| 血糖 | mmol/L | 0 - 50 | 一位小数 |
| 收缩压（高压） | mmHg | 60 - 250 | 整数 |
| 舒张压（低压） | mmHg | 40 - 200 | 整数 |
| 心率 | 次/分 | 20 - 300 | 整数 |
| 体温 | °C | 30 - 45 | 一位小数 |
| 体重 | kg | 20 - 300 | 一位小数 |
| BMI | - | 0 - 100 | 一位小数 |

**保健指标**:

| 指标 | 单位 | 有效范围 |
|------|------|---------|
| 睡眠时长 | 小时 | 0 - 24 |
| 步数 | 步 | 0 - 100,000 |
| 饮水量 | ml | 0 - 10,000 |
| 心情评分 | 分 | 1 - 10 |
| 精力评分 | 分 | 1 - 10 |
| 运动时长 | 分钟 | 0 - 1,440 |

> **血压录入特别说明**: 血压需同时输入收缩压和舒张压，系统自动保存为两条记录。

### 5.2 AI 智能录入

#### 5.2.1 操作步骤

1. 进入「智能录入」页面 → 找到「AI 智能录入」卡片
2. 在文本框中输入自然语言描述
3. 点击「智能解析」按钮
4. 查看解析结果，勾选/取消需要保存的指标
5. 点击「确认录入」保存

#### 5.2.2 支持的输入格式

```
# 单指标
"今天血糖5.6"
"心率72"
"体重65公斤"

# 多指标
"今天血糖5.6，心率72，走了8000步"
"早上血压120/80，血糖空腹6.2"

# 带时间描述
"昨天睡眠7小时"
"前天步数10000步"
```

#### 5.2.3 解析置信度

解析结果会标注置信度，帮助判断准确性：

| 置信度 | 颜色标识 | 建议 |
|--------|---------|------|
| 90%+ | 绿色 | 高可信，可直接保存 |
| 70%-90% | 黄色 | 建议核对后保存 |
| <70% | 红色 | 建议手动修正后保存 |

#### 5.2.4 使用限制

- 登录用户：每日 **20次**
- 界面显示「今日剩余」次数
- 次数每日零点重置

#### 5.2.5 解析历史管理

系统保存每次 AI 解析的完整记录，支持：
- 查看历史解析列表（分页浏览）
- 查看单条解析详情（含原始文本和解析结果）
- 删除历史记录

### 5.3 OCR 图片识别录入

#### 5.3.1 操作步骤

1. 进入「智能录入」页面 → 找到「OCR 图片识别」卡片
2. 选择识别类型：
   - **通用健康图片**: 自动识别图片类型
   - **体检报告**: 医院体检报告单、血常规、生化检查
   - **药品标签**: 药品包装或说明书
   - **营养标签**: 食品营养成分表
3. 拖拽或点击上传图片
4. 点击「开始识别」
5. 查看识别结果：
   - 勾选/取消识别的指标
   - 点击「原始识别文本」查看完整 OCR 内容
6. 点击「确认录入」保存数据

#### 5.3.2 图片要求

| 项目 | 要求 |
|------|------|
| 格式 | JPG / JPEG / PNG / WebP |
| 大小 | 最大 10MB |
| 画质 | 清晰、文字水平放置、光线充足 |
| 内容 | 支持印刷体，手写体效果有限 |

#### 5.3.3 识别流程

```
上传图片 → 百度 OCR 识别文字 → AI 解析结构化数据 → 用户确认 → 写入健康指标
```

#### 5.3.4 OCR 历史管理

系统保存每次 OCR 识别的完整记录，支持：
- 分页浏览识别历史
- 查看单条记录详情（含原图和识别结果）
- 删除历史记录

### 5.4 穿戴设备同步

#### 5.4.1 支持的设备平台

| 平台 | 设备类型 | 可同步数据 | 状态 |
|------|---------|-----------|------|
| 华为健康 | 华为手环/手表 | 心率、血压、步数、睡眠 | 已支持 |
| 小米健康 | 小米手环/手表 | 心率、步数、睡眠、体重 | 已支持 |
| 微信运动 | 手机计步 | 步数 | 规划中 |
| Apple 健康 | iPhone/Apple Watch | 多种指标 | 规划中 |

#### 5.4.2 连接设备

1. 进入「智能录入」页面 → 找到「穿戴设备同步」卡片
2. 选择目标平台 → 点击「连接」
3. 系统检查平台可用性（确认 OAuth 已配置）
4. 跳转至对应平台的 OAuth 授权页面
5. 用户授权后自动返回，连接状态变为「已连接」

> **授权说明**: 首次连接需要跳转到对应平台授权。系统会先检查平台配置状态，未配置的平台会提示管理员配置。授权后系统可读取设备数据，可随时断开连接撤销授权。

#### 5.4.3 同步数据

**单设备同步**: 找到已连接设备 → 点击「同步」→ 等待完成

**全部同步**: 点击「同步全部」→ 系统依次同步所有已连接设备

**同步状态说明**:

| 状态 | 含义 |
|------|------|
| 已连接 | 设备已授权，可正常同步 |
| 未连接 | 设备未授权，需先连接 |
| 同步中 | 正在获取数据 |
| 同步成功 | 数据已导入系统 |
| 同步失败 | 请检查网络或重新授权 |

#### 5.4.4 同步历史

系统记录每次同步操作的详细信息：
- 同步时间、平台、状态
- 同步的数据条数
- 分页浏览历史记录

#### 5.4.5 同步频率限制

| 操作 | 限制 |
|------|------|
| 手动同步 | 每设备每分钟 1 次 |
| Token 刷新 | 每小时 5 次 |

## 6. 健康指标管理

**操作路径**: 左侧导航栏 → 健康管理 → 健康指标 (`/health/metrics`)

### 6.1 指标列表

- 支持分页浏览所有历史健康指标
- 按指标类型筛选（血糖、血压、心率等）
- 按时间范围查询
- 显示指标名称、数值、单位、记录日期、趋势标识

### 6.2 指标操作

| 操作 | 说明 |
|------|------|
| 查看详情 | 查看指标详细信息和趋势 |
| 编辑 | 修改数值、日期等信息 |
| 删除 | 删除指标记录（操作会记入删除日志） |

### 6.3 趋势标识

每条记录带有趋势标记：

| 趋势 | 含义 |
|------|------|
| NORMAL | 指标在正常范围内 |
| HIGH | 指标偏高 |
| LOW | 指标偏低 |

### 6.4 指标分类

系统将指标分为两大类：

- **HEALTH（健康指标）**: 血糖、血压、心率、体温、体重、BMI
- **WELLNESS（保健指标）**: 睡眠、步数、运动、饮水、心情、精力

## 7. 实时监控

**操作路径**: 左侧导航栏 → 实时监控 (`/realtime/monitor`)

### 7.1 功能说明

实时监控模块提供健康指标的即时录入、最新值展示和历史趋势图表。适合需要频繁监测的场景（如运动后心率、饭后血糖）。

### 7.2 操作步骤

1. 选择要录入的指标类型（心率、血压、血糖等）
2. 输入数值
3. 点击「提交」
4. 系统自动：
   - 保存实时指标
   - 执行数据去重（5分钟内同一指标不重复记录）
   - 触发预警规则评估

### 7.3 最新指标展示

页面以卡片形式展示用户每种指标的最新值，包含：
- 指标名称和数值
- 单位
- 录入时间

### 7.4 趋势图表

- 支持查看特定指标在指定时间段内的趋势（如最近 24 小时）
- 使用 ECharts 图表动态渲染
- 可选择不同时间跨度：6小时、12小时、24小时、7天

### 7.5 数据去重规则

同一用户、同一指标在 **5分钟** 时间窗口内不重复记录，防止设备同步或误操作产生冗余数据。

## 8. 健康评分与报告

**操作路径**: 健康管理 → 健康评分 (`/health/score`)

### 8.1 健康评分

#### 8.1.1 评分机制

系统根据五个维度计算综合健康评分（满分100分）：

| 维度 | 权重 | 评估内容 |
|------|------|---------|
| 基础指标 | 30% | BMI、血压、血糖等核心生理指标 |
| 生活习惯 | 25% | 睡眠、运动、饮水等日常行为 |
| 健康档案 | 20% | 年龄、性别、家族病史等基本信息 |
| 指标趋势 | 15% | 近期指标变化方向和幅度 |
| 预警记录 | 10% | 预警数量和严重程度 |

#### 8.1.2 评分等级

| 分数范围 | 等级 | 评价 | 色标 |
|----------|------|------|------|
| 90 - 100 | EXCELLENT | 优秀 | 深绿 |
| 75 - 89 | GOOD | 良好 | 浅绿 |
| 60 - 74 | FAIR | 一般 | 黄色 |
| 0 - 59 | POOR | 较差 | 红色 |

#### 8.1.3 页面展示

- **综合评分环形图**: 直观显示总评分和等级
- **各维度评分**: 显示每个维度的得分和占比
- **评分因素标签**: 展示影响评分的关键因素
- **改进建议**: 基于薄弱维度提供个性化健康建议

#### 8.1.4 评分历史

系统记录历史评分数据，支持：
- 查看近 30 天（可自定义天数）的评分变化曲线
- 追踪评分趋势，了解健康状况变化方向

#### 8.1.5 刷新机制

- 评分结果缓存 **1小时**
- 录入新指标后自动更新评分
- 用户可手动点击「重新计算」强制刷新评分

### 8.2 AI 健康报告

#### 8.2.1 功能说明

系统基于用户健康数据，通过 AI 自动生成详细的健康分析报告。报告内容包括：

| 报告部分 | 说明 |
|---------|------|
| 用户信息 | 性别、年龄、身高、体重、BMI |
| 总体评分 | 综合健康评分与评分等级 |
| 维度分析 | 各健康维度的评分、状态和描述 |
| 风险提示 | 按风险等级列出的健康风险项，含建议措施 |
| 改善建议 | 按优先级排序的个性化改善建议和行动方案 |
| 总结 | 健康状况综合描述 |

#### 8.2.2 报告操作

| 操作 | 说明 |
|------|------|
| 查看最新报告 | 自动获取最近一次生成的报告（优先使用缓存） |
| 生成新报告 | 强制重新生成报告，反映最新健康数据 |
| 查看历史报告 | 浏览过去生成的所有报告 |
| 查看指定报告 | 通过报告 ID 查看特定报告 |

#### 8.2.3 报告生成流程

```
用户请求 → 聚合健康数据 → 注入用户档案 → AI 分析生成 → 结构化报告 → 缓存结果
```

> **注意**: 报告生成基于当前健康数据快照，数据更新后建议重新生成以获得最新分析。

## 9. 智能预警系统

**操作路径**: 健康管理 → 健康预警 (`/health/alerts`)

### 9.1 预警机制

当健康指标超出设定阈值时，系统自动生成预警通知。预警流程：

```
指标录入 → 规则引擎评估 → 预警去重 → 生成预警 → AI 分析 → 多渠道推送
```

### 9.2 预警级别

| 级别 | 图标 | 含义 | 典型场景 |
|------|------|------|---------|
| CRITICAL 严重 | 红色 | 需立即关注 | 心率 > 120 bpm |
| WARNING 警告 | 黄色 | 需要注意 | 血压 > 140 mmHg |
| INFO 提示 | 蓝色 | 健康建议 | BMI 偏高 |
| TREND 趋势 | 紫色 | 趋势变化 | 血糖连续上升 |
| RECOVERY 恢复 | 绿色 | 恢复正常 | 血压恢复正常 |

### 9.3 预警规则（默认阈值）

| 指标 | 警告上限 | 严重上限 | 警告下限 | 严重下限 |
|------|----------|----------|----------|----------|
| 心率 (bpm) | 100 | 120 | 50 | 40 |
| 收缩压 (mmHg) | 140 | 160 | 90 | 80 |
| 舒张压 (mmHg) | 90 | 100 | 60 | 50 |
| 血糖 (mmol/L) | 7.0 | 11.1 | 3.9 | 3.0 |
| BMI | 28 | 35 | 18.5 | 16 |
| 体温 (°C) | 37.5 | 38.5 | 36.0 | 35.5 |

### 9.4 个性化阈值配置

**操作路径**: 健康管理 → 阈值配置 (`/health/thresholds`)

用户可为每种指标设置自己的预警阈值，覆盖系统默认值：

1. 选择指标类型
2. 设置警告上限/下限、严重上限/下限
3. 点击「保存」

> **优先级**: 用户自定义阈值 > 系统默认阈值

### 9.5 预警操作

| 操作 | 说明 |
|------|------|
| 查看列表 | 按时间倒序查看所有预警，支持按类型和已读/未读筛选 |
| 标记已读 | 确认已查看该预警 |
| 确认预警 | 标记已处理该预警（与已读分开管理） |
| 全部已读 | 一键标记所有预警为已读 |
| 删除预警 | 删除单条预警记录 |

### 9.6 预警统计

系统提供预警数据分析功能：
- **总体统计**: 各级别预警数量分布
- **趋势分析**: 特定指标预警的频率趋势
- **早期预警**: 基于指标趋势的预测性预警提示

### 9.7 智能去重

同一用户、同一指标、同一类型的预警，**30分钟内** 只生成一次，避免重复打扰。

### 9.8 AI 预警分析

系统自动调用 AI 对预警进行简要分析，提供：
- 预警原因分析
- 个性化健康建议
- 是否需要就医提示

### 9.9 推送渠道

预警通知支持多渠道推送：

| 渠道 | 说明 | 配置要求 |
|------|------|---------|
| 站内通知 | 系统内弹窗/列表展示 | 默认启用 |
| 邮件通知 | 发送至注册邮箱 | 需配置 SMTP |
| 企业微信 | 推送到企业微信群 | 需配置 Webhook |
| 飞书 | 推送到飞书群 | 需配置 Webhook |

> **推送配置路径**: 个人中心 → 推送设置 (`/user/push-settings`)

  - 日常饮食规划

## 16. 安全与隐私

### 16.1 安全措施

| 安全项 | 措施 |
|--------|------|
| 传输加密 | 全站 HTTPS 加密通信 |
| 密码保护 | BCrypt 算法加密存储 |
| 身份认证 | JWT 令牌认证机制（7天有效期） |
| Token 安全 | 设备 OAuth Token 使用 AES-GCM 加密存储 |
| 输入防护 | SQL 注入防护、XSS 过滤（DOMPurify）、CSRF 保护 |
| 内容过滤 | AI 输入/输出双向内容安全过滤 |
| 限流保护 | API 请求频率限制（AI 对话、设备同步等），防止滥用 |
| 操作审计 | 关键操作（删除等）记录审计日志 |

### 16.2 数据隐私

- 个人健康数据仅用户本人可见
- AI 对话记录仅用于对话上下文，不用于其他目的
- 设备同步数据经加密传输和存储
- 用户可随时删除自己的健康数据

## 17. 常见问题 FAQ

### 17.1 账户与登录

**Q: 忘记密码怎么办？**
A: 请联系系统管理员重置密码。

**Q: 登录后提示会话过期？**
A: 登录凭证有效期为 7 天，超时后请重新登录。

**Q: 无法登录，提示密码错误？**
A: 请检查用户名是否正确、密码大小写、输入法是否为英文模式。

### 17.2 数据录入

**Q: 数值输入提示超出范围？**
A: 请参照各指标有效范围（参见第 5.1.2 节或附录 A）。

**Q: 血压录入保存失败？**
A: 血压需同时输入收缩压和舒张压，请确保两项都已填写。

**Q: 可以补录过去的健康数据吗？**
A: 可以。在录入对话框中选择历史日期即可。

### 17.3 AI 与 OCR

**Q: AI 解析失败？**
A: 可能原因：描述不够清晰（建议用"血糖5.6"而非"血糖正常"）、服务暂时不可用、每日次数用完。

**Q: OCR 识别不准确？**
A: 请确保图片清晰、文字水平、光线充足。主要支持印刷体，手写体效果有限。

**Q: AI 对话提示次数用完？**
A: 登录用户每日 20 次对话额度，次日零点自动重置。

**Q: AI 回复内容格式显示异常？**
A: AI 回复支持 Markdown 格式，如果显示异常请刷新页面。如持续异常请联系管理员。

### 17.4 设备同步

**Q: 授权页面打不开？**
A: 请检查浏览器是否阻止了弹出窗口，尝试允许弹出窗口后重试。

**Q: 同步失败？**
A: 可能原因：授权过期（断开后重新连接）、网络问题、设备平台服务异常。

**Q: 同步的数据在哪里查看？**
A: 同步成功后数据自动保存到对应的健康指标中。

**Q: 平台提示「未配置」怎么办？**
A: 设备平台需要管理员先配置 OAuth 凭据。请联系管理员配置后再连接。

### 17.5 预警

**Q: 为什么没有收到预警？**
A: 请检查：1) 是否已设置阈值（默认阈值或自定义阈值）；2) 推送渠道是否已启用。

**Q: 预警太频繁？**
A: 系统已在 30 分钟内对相同预警去重。您还可以调高个性化阈值来减少预警触发。

**Q: 如何测试推送是否正常？**
A: 在推送设置页面，点击对应渠道的「测试推送」按钮，系统会发送一条测试消息。

### 17.6 主题与界面

**Q: 如何切换暗色模式？**
A: 点击顶部导航栏右侧的月亮图标即可切换。也可以在设置中选择「跟随系统」自动适配。

**Q: 浮动球挡住了页面内容怎么办？**
A: 浮动球支持拖拽，可以拖到页面任意位置。
