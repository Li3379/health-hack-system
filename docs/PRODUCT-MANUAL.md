## 1. 产品简介

### 1.1 产品定位

**HHS（Health Hack System）** 是一款面向个人用户的智能健康管理平台，集健康数据追踪、AI 智能分析、实时监控、预警通知和可穿戴设备联动于一体，帮助用户全面掌握自身健康状况。

### 1.2 核心功能一览

| 功能模块 | 说明 | 入口路径 |
|---------|------|---------|
| 仪表盘 | 健康数据综合概览与快捷操作 | 首页 `/dashboard` |
| 数据录入 | 快捷/AI/OCR/设备同步四种录入方式 | `/data-input` |
| 健康指标 | 历史指标查询、编辑与趋势分析 | `/health/metrics` |
| 实时监控 | 实时指标录入与动态趋势图 | `/realtime/monitor` |
| 健康评分 | 多维度综合健康评估 | `/health/score` |
| 智能预警 | 多级预警通知与个性化阈值 | `/health/alerts` |
| AI 顾问 | 智能健康咨询对话 | `/ai/chat` |
| 预防保健 | 健康档案与风险评估 | `/prevention/profile` |
| 保健指标 | 睡眠、运动、心情等日常保健追踪 | `/wellness/dashboard` |
| 筛查管理 | 体检报告管理与异常追踪 | `/screening/list` |
| 个人中心 | 个人资料与推送设置 | `/user/profile` |

### 1.3 系统架构概览

```
┌───────────────────────────────────────────────────────┐
│                    用户（浏览器/移动端）                │
└─────────────────────┬─────────────────────────────────┘
                      │ HTTPS / WebSocket
┌─────────────────────▼─────────────────────────────────┐
│              Vue 3 前端 (Element Plus UI)              │
├───────────────────────────────────────────────────────┤
│              Spring Boot 后端 (REST API)               │
├────────┬────────┬────────┬────────┬──────────────────┤
│ MySQL  │ Redis  │ 通义千问 │ 百度OCR │ 华为/小米 OAuth │
└────────┴────────┴────────┴────────┴──────────────────┘
```

## 3. 账户管理

### 3.1 注册新账户

**操作路径**: 系统首页 → 点击「立即注册」

**步骤**:

1. 填写注册信息：

   | 字段 | 要求 | 说明 |
   |------|------|------|
   | 用户名 | 4-32个字符 | 用于登录，注册后不可修改 |
   | 密码 | 6-64个字符 | 建议包含大小写字母、数字和特殊字符 |
   | 确认密码 | 与密码一致 | 再次输入确认 |
   | 邮箱 | 可选 | 格式如 `user@example.com`，用于通知 |

2. 点击「立即注册」完成注册
3. 注册成功后自动跳转至登录页

> **注意**: 用户名一旦注册不可修改。请妥善保管密码，建议使用强密码。

### 3.2 登录系统

**操作路径**: 系统首页

1. 输入用户名和密码
2. 点击「登录」按钮
3. 登录成功后进入仪表盘

**会话说明**:
- 登录凭证有效期为 **7天**，期间无需重复登录
- Token 过期后需重新登录
- 同一账户不支持多端同时在线

### 3.3 退出登录

**操作路径**: 右上角用户头像 → 退出登录

退出后系统清除本地登录凭证，需重新登录。

## 5. 健康数据录入

**操作路径**: 左侧导航栏 → 数据录入 (`/data-input`)

HHS 提供 **四种** 数据录入方式，覆盖不同场景需求：

| 录入方式 | 适用场景 | 便捷度 | 准确度 |
|---------|---------|--------|--------|
| 快捷录入 | 手动输入单个指标 | ★★★ | ★★★★★ |
| AI 智能录入 | 自然语言批量描述 | ★★★★★ | ★★★★ |
| OCR 识别 | 纸质报告/化验单拍照 | ★★★★ | ★★★ |
| 设备同步 | 可穿戴设备自动导入 | ★★★★★ | ★★★★★ |

### 5.1 快捷录入

#### 5.1.1 操作步骤

1. 进入「数据录入」页面 → 点击「快捷录入」卡片
2. 点击需要录入的指标按钮（如：血糖、心率、血压等）
3. 在弹出的对话框中：
   - 输入数值（系统实时验证范围）
   - 选择记录日期（默认当天，支持补录历史数据）
4. 点击「保存」完成录入

#### 5.1.2 支持的指标

**健康指标**:

| 指标 | 单位 | 有效范围 | 精度 |
|------|------|---------|------|
| 血糖 | mmol/L | 0 - 50 | 一位小数 |
| 收缩压（高压） | mmHg | 60 - 250 | 整数 |
| 舒张压（低压） | mmHg | 40 - 200 | 整数 |
| 心率 | 次/分 | 20 - 300 | 整数 |
| 体温 | °C | 30 - 45 | 一位小数 |
| 体重 | kg | 20 - 300 | 一位小数 |
| BMI | - | 0 - 100 | 一位小数 |

**保健指标**:

| 指标 | 单位 | 有效范围 |
|------|------|---------|
| 睡眠时长 | 小时 | 0 - 24 |
| 步数 | 步 | 0 - 100,000 |
| 饮水量 | ml | 0 - 10,000 |
| 心情评分 | 分 | 1 - 10 |
| 精力评分 | 分 | 1 - 10 |
| 运动时长 | 分钟 | 0 - 1,440 |

> **血压录入特别说明**: 血压需同时输入收缩压和舒张压，系统自动保存为两条记录。

### 5.2 AI 智能录入

#### 5.2.1 操作步骤

1. 进入「数据录入」页面 → 找到「AI 智能录入」卡片
2. 在文本框中输入自然语言描述
3. 点击「智能解析」按钮
4. 查看解析结果，勾选/取消需要保存的指标
5. 点击「确认录入」保存

#### 5.2.2 支持的输入格式

```
# 单指标
"今天血糖5.6"
"心率72"
"体重65公斤"

# 多指标
"今天血糖5.6，心率72，走了8000步"
"早上血压120/80，血糖空腹6.2"

# 带时间描述
"昨天睡眠7小时"
"前天步数10000步"
```

#### 5.2.3 解析置信度

解析结果会标注置信度，帮助判断准确性：

| 置信度 | 颜色标识 | 建议 |
|--------|---------|------|
| 90%+ | 绿色 | 高可信，可直接保存 |
| 70%-90% | 黄色 | 建议核对后保存 |
| <70% | 红色 | 建议手动修正后保存 |

#### 5.2.4 使用限制

- 登录用户：每日 **20次**
- 界面显示「今日剩余」次数
- 次数每日零点重置

### 5.3 OCR 图片识别录入

#### 5.3.1 操作步骤

1. 进入「数据录入」页面 → 找到「OCR 图片识别」卡片
2. 选择识别类型：
   - **体检报告**: 医院体检报告单、血常规、生化检查
   - **药品标签**: 药品包装或说明书
   - **营养标签**: 食品营养成分表
3. 拖拽或点击上传图片
4. 点击「开始识别」
5. 查看识别结果：
   - 勾选/取消识别的指标
   - 点击「原始识别文本」查看完整 OCR 内容
6. 点击「确认录入」保存数据

#### 5.3.2 图片要求

| 项目 | 要求 |
|------|------|
| 格式 | JPG / JPEG / PNG / WebP |
| 大小 | 最大 10MB |
| 画质 | 清晰、文字水平放置、光线充足 |
| 内容 | 支持印刷体，手写体效果有限 |

#### 5.3.3 识别流程

```
上传图片 → 百度 OCR 识别文字 → AI 解析结构化数据 → 用户确认 → 写入健康指标
```

### 5.4 穿戴设备同步

#### 5.4.1 支持的设备平台

| 平台 | 设备类型 | 可同步数据 | 状态 |
|------|---------|-----------|------|
| 华为健康 | 华为手环/手表 | 心率、血压、步数、睡眠 | 已支持 |
| 小米健康 | 小米手环/手表 | 心率、步数、睡眠、体重 | 已支持 |
| 微信运动 | 手机计步 | 步数 | 规划中 |
| Apple 健康 | iPhone/Apple Watch | 多种指标 | 规划中 |

#### 5.4.2 连接设备

1. 进入「数据录入」页面 → 找到「穿戴设备同步」卡片
2. 选择目标平台 → 点击「连接」
3. 系统跳转至对应平台的 OAuth 授权页面
4. 用户授权后自动返回，连接状态变为「已连接」

> **授权说明**: 首次连接需要跳转到对应平台授权。授权后系统可读取设备数据。可随时断开连接撤销授权。

#### 5.4.3 同步数据

**单设备同步**: 找到已连接设备 → 点击「同步」→ 等待完成

**全部同步**: 点击「同步全部」→ 系统依次同步所有已连接设备

**同步状态说明**:

| 状态 | 含义 |
|------|------|
| 已连接 | 设备已授权，可正常同步 |
| 未连接 | 设备未授权，需先连接 |
| 同步中 | 正在获取数据 |
| 同步成功 | 数据已导入系统 |
| 同步失败 | 请检查网络或重新授权 |

#### 5.4.4 同步频率限制

| 操作 | 限制 |
|------|------|
| 手动同步 | 每设备每分钟 1 次 |
| Token 刷新 | 每小时 5 次 |

## 6. 健康指标管理

**操作路径**: 左侧导航栏 → 健康管理 → 健康指标 (`/health/metrics`)

### 6.1 指标列表

- 支持分页浏览所有历史健康指标
- 按指标类型筛选（血糖、血压、心率等）
- 按时间范围查询
- 显示指标名称、数值、单位、记录日期、趋势标识

### 6.2 指标操作

| 操作 | 说明 |
|------|------|
| 查看详情 | 查看指标详细信息和趋势 |
| 编辑 | 修改数值、日期等信息 |
| 删除 | 删除指标记录（操作会记入删除日志） |

### 6.3 趋势标识

每条记录带有趋势标记：

| 趋势 | 含义 |
|------|------|
| NORMAL | 指标在正常范围内 |
| HIGH | 指标偏高 |
| LOW | 指标偏低 |

### 6.4 指标分类

系统将指标分为两大类：

- **HEALTH（健康指标）**: 血糖、血压、心率、体温、体重、BMI
- **WELLNESS（保健指标）**: 睡眠、步数、运动、饮水、心情、精力

## 7. 实时监控

**操作路径**: 左侧导航栏 → 实时监控 (`/realtime/monitor`)

### 7.1 功能说明

实时监控模块提供健康指标的即时录入、最新值展示和历史趋势图表。适合需要频繁监测的场景（如运动后心率、饭后血糖）。

### 7.2 操作步骤

1. 选择要录入的指标类型（心率、血压、血糖等）
2. 输入数值
3. 点击「提交」
4. 系统自动：
   - 保存实时指标
   - 执行数据去重（5分钟内同一指标不重复记录）
   - 触发预警规则评估

### 7.3 最新指标展示

页面以卡片形式展示用户每种指标的最新值，包含：
- 指标名称和数值
- 单位
- 录入时间

### 7.4 趋势图表

- 支持查看特定指标在指定时间段内的趋势（如最近 24 小时）
- 使用 ECharts 图表动态渲染
- 可选择不同时间跨度：6小时、12小时、24小时、7天

### 7.5 数据去重规则

同一用户、同一指标在 **5分钟** 时间窗口内不重复记录，防止设备同步或误操作产生冗余数据。

## 8. 健康评分

**操作路径**: 健康管理 → 健康评分 (`/health/score`)

### 8.1 评分机制

系统根据五个维度计算综合健康评分（满分100分）：

| 维度 | 权重 | 评估内容 |
|------|------|---------|
| 基础指标 | 30% | BMI、血压、血糖等核心生理指标 |
| 生活习惯 | 25% | 睡眠、运动、饮水等日常行为 |
| 健康档案 | 20% | 年龄、性别、家族病史等基本信息 |
| 指标趋势 | 15% | 近期指标变化方向和幅度 |
| 预警记录 | 10% | 预警数量和严重程度 |

### 8.2 评分等级

| 分数范围 | 等级 | 评价 | 色标 |
|----------|------|------|------|
| 90 - 100 | EXCELLENT | 优秀 | 深绿 |
| 75 - 89 | GOOD | 良好 | 浅绿 |
| 60 - 74 | FAIR | 一般 | 黄色 |
| 0 - 59 | POOR | 较差 | 红色 |

### 8.3 页面展示

- **综合评分环形图**: 直观显示总评分和等级
- **各维度评分**: 显示每个维度的得分和占比
- **改进建议**: 基于薄弱维度提供个性化健康建议

### 8.4 刷新机制

- 评分结果缓存 **1小时**
- 录入新指标后自动更新评分
- 用户可手动刷新

## 9. 智能预警系统

**操作路径**: 健康管理 → 健康预警 (`/health/alerts`)

### 9.1 预警机制

当健康指标超出设定阈值时，系统自动生成预警通知。预警流程：

```
指标录入 → 规则引擎评估 → 预警去重 → 生成预警 → AI 分析 → 多渠道推送
```

### 9.2 预警级别

| 级别 | 图标 | 含义 | 典型场景 |
|------|------|------|---------|
| CRITICAL 严重 | 🔴 | 需立即关注 | 心率 > 120 bpm |
| WARNING 警告 | 🟡 | 需要注意 | 血压 > 140 mmHg |
| INFO 提示 | 🔵 | 健康建议 | BMI 偏高 |
| TREND 趋势 | 🟣 | 趋势变化 | 血糖连续上升 |
| RECOVERY 恢复 | 🟢 | 恢复正常 | 血压恢复正常 |

### 9.3 预警规则（默认阈值）

| 指标 | 警告上限 | 严重上限 | 警告下限 | 严重下限 |
|------|----------|----------|----------|----------|
| 心率 (bpm) | 100 | 120 | 50 | 40 |
| 收缩压 (mmHg) | 140 | 160 | 90 | 80 |
| 舒张压 (mmHg) | 90 | 100 | 60 | 50 |
| 血糖 (mmol/L) | 7.0 | 11.1 | 3.9 | 3.0 |
| BMI | 28 | 35 | 18.5 | 16 |

### 9.4 个性化阈值配置

**操作路径**: 健康管理 → 阈值配置 (`/health/thresholds`)

2. 设置警告上限/下限、严重上限/下限
3. 点击「保存」

## 15. 安全与隐私

### 15.1 安全措施

| 安全项 | 措施 |
|--------|------|
| 传输加密 | 全站 HTTPS 加密通信 |

- AI 对话记录仅用于对话上下文，不用于其他目的
- 设备同步数据经加密传输和存储
- 用户可随时删除自己的健康数据

## 16. 常见问题 FAQ

### 16.1 账户与登录

**Q: 忘记密码怎么办？**
A: 请联系系统管理员重置密码。

**Q: 登录后提示会话过期？**
A: 登录凭证有效期为 7 天，超时后请重新登录。

**Q: 无法登录，提示密码错误？**
A: 请检查用户名是否正确、密码大小写、输入法是否为英文模式。

### 16.2 数据录入

**Q: 数值输入提示超出范围？**
A: 请参照各指标有效范围（参见第 5.1.2 节或附录 A）。

**Q: 血压录入保存失败？**
A: 血压需同时输入收缩压和舒张压，请确保两项都已填写。

**Q: 可以补录过去的健康数据吗？**
A: 可以。在录入对话框中选择历史日期即可。

### 16.3 AI 与 OCR

**Q: AI 解析失败？**
A: 可能原因：描述不够清晰（建议用"血糖5.6"而非"血糖正常"）、服务暂时不可用、每日次数用完。

**Q: OCR 识别不准确？**
A: 请确保图片清晰、文字水平、光线充足。主要支持印刷体，手写体效果有限。

### 16.4 设备同步

**Q: 授权页面打不开？**
A: 请检查浏览器是否阻止了弹出窗口，尝试允许弹出窗口后重试。

**Q: 同步失败？**
A: 可能原因：授权过期（断开后重新连接）、网络问题、设备平台服务异常。

**Q: 同步的数据在哪里查看？**
A: 同步成功后数据自动保存到对应的健康指标中。

### 16.5 预警

**Q: 为什么没有收到预警？**
A: 请检查：1) 是否已设置阈值（默认阈值或自定义阈值）；2) 推送渠道是否已启用。

**Q: 预警太频繁？**
A: 系统已在 30 分钟内对相同预警去重。您还可以调高个性化阈值来减少预警触发。
